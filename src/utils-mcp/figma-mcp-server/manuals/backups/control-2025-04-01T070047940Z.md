### 깃 배포 프로세스

1. 변경사항 스테이징

```bash
git add .
```

2. 변경사항 커밋

```bash
git commit -m "feat: 기능 추가 또는 수정 내용"
```

3. 원격 저장소 업데이트

```bash
git pull origin main
```

4. 변경사항 푸시

```bash
git push origin main
```

### 깃 브랜치 전략

1. 브랜치 종류

   - `main`: 프로덕션 환경
   - `develop`: 개발 환경
   - `feature/*`: 새로운 기능 개발
   - `hotfix/*`: 긴급 버그 수정

2. 브랜치 생성

```bash
git checkout -b feature/new-feature
```

3. 브랜치 병합

```bash
git checkout main
git merge feature/new-feature
```
